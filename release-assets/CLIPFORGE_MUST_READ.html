ClipForge 内测版

ClipForge 全功能使用说明

如果 macOS 提示“无法验证开发者”、“已损坏，应该移到废纸篓”或“Apple 无法检查其是否包含恶意软件”， 这通常不是应用真的损坏，而是 Gatekeeper 在拦截一个尚未完成 Apple 公证的内测包。

核心定位 先做完整的跨平台剪贴板工具，再扩展搜索、归档和 MCP 能力。
当前形态 独立悬浮面板 + 设置窗口 + 本地 SQLite 持久化 + 全局快捷键。
安装口径 这是内测包，不是 App Store 公证版；首次运行需要按本文说明处理。

当前版本包含的功能

剪贴板历史 自动记录文本内容，按时间倒序浏览，并支持本地持久化。
快速唤起面板 通过全局快捷键或托盘打开悬浮面板，进行搜索、复制、粘贴和选择。
本地搜索 支持按关键词、类型、标签和收藏状态筛选历史内容。
收藏与归档 支持收藏常用条目、软删除、恢复和彻底删除。
片段与工作区结构 当前版本已经包含片段、视图切换和面板内工作区承载结构。
本地服务接口 应用内已经暴露标准化服务接口，供前端、MCP 和后续外部能力复用。

推荐安装方式

  1. 打开 DMG，把 ClipForge.app 拖到 Applications。
  2. 不要直接双击，先在 Applications 里对 ClipForge.app 右键。
  3. 选择 “打开”，再在系统弹窗里点击 “打开” 或 “仍要打开”。

完成一次后，后续通常就可以直接双击启动。

如果仍被拦截

可以在终端执行下面这条命令，移除下载隔离属性：

xattr -r -d com.apple.quarantine /Applications/ClipForge.app

如果你的安装位置不是 Applications，请把路径替换成实际路径。

首次运行还需要的权限

  • 辅助功能权限：用于贴近当前输入区域定位悬浮面板，以及执行粘贴动作。
  • 全局快捷键权限：用于系统级唤起 ClipForge。

推荐使用流程

  1. 启动后先完成辅助功能授权，确保悬浮面板定位和粘贴能力可用。
  2. 用快捷键或托盘唤起主面板，确认能看到剪贴板历史。
  3. 复制一段文本，回到面板检查是否已写入历史。
  4. 在搜索框中输入关键字，验证筛选、复制和粘贴动作。
  5. 再打开设置窗口，核对配置路径、数据库路径和日志路径是否正常。

当前内测重点

  • 悬浮面板在不同显示器、不同激活窗口下的定位是否稳定。
  • 复制后历史是否稳定入库，是否出现漏记、重复或回写污染。
  • 快捷键、托盘唤起、复制、粘贴、删除和恢复链路是否顺畅。
  • 权限未授权、首次安装、重新打开后的行为是否可理解。

不推荐的做法

不要为了安装内测版而全局关闭 macOS 安全机制，例如：

sudo spctl --master-disable

这会降低整台机器的安全性。优先使用右键打开，或者只移除 ClipForge 的隔离属性。

已知边界

  • 当前分发包仍属于内测包，没有 Apple Developer 签名和 notarization。
  • 首次安装时遇到 Gatekeeper 拦截属于预期现象，需要按本文处理。
  • 如果面板位置、活动窗口或剪贴板监听异常，请优先反馈当时的应用场景和权限状态。
