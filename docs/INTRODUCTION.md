# ClipForge 介绍

ClipForge 是一个跨平台剪贴板工作台，目标是先把 Clipy 的核心体验做扎实：快速唤起、历史记录、搜索、收藏、批量操作和复制回写。它不是一个 AI 平台，也不是复杂知识库；主体验必须保持轻、快、稳定。

## 为什么做 ClipForge

高频剪贴板工具的关键不是功能堆叠，而是低打扰：

- 快捷键触发后快速出现。
- 面板尽量小，不遮挡当前工作区。
- 内容可以马上搜索、筛选和复制。
- 长期保存的数据必须在本机持久化。
- 后续自动化或 Agent 接入必须走标准接口，不影响普通用户体验。

ClipForge 使用 Tauri v2 承接系统能力，用 React 承接快速迭代的交互层，用 SQLite 承接本地长期存储。

## 当前能力

- 文本剪贴板采集、去重和持久化。
- macOS 默认 `Control+V` 快捷唤起。
- 系统状态栏菜单触发快速面板。
- 历史和收藏视图。
- 搜索、类型 Tag、保存搜索规则。
- 多选、批量删除、聚合复制。
- 软删除和定期清理。
- Markdown 识别和详情预览。
- 本地 JSON5 配置文件。
- SQLite 本地数据库。
- 面向同步、导入导出和 MCP 的 TypeScript 服务契约。

## 设计原则

1. 高频动作优先：复制、搜索、删除、收藏必须比管理功能更醒目。
2. 本地优先：剪贴板数据默认留在用户机器上。
3. 可恢复：删除默认软删除，清理策略可配置。
4. 可扩展：同步和 MCP 通过稳定服务接口进入，不直接穿透 UI 状态。
5. 跨平台：平台差异隔离在 Tauri/Rust 层。

## 项目状态

ClipForge 目前处于早期开发阶段，接口和 UI 仍在快速调整。仓库会优先稳定剪贴板核心体验，再扩展发布构建、同步和 MCP server。
