# 提案：初次引导程序迁移至设置面板

## 优先级

P1。该优化直接改善用户体验：悬浮面板空间有限，应该聚焦剪贴板历史、搜索、复制等核心功能；引导、权限配置和功能介绍适合在设置面板这种大空间中完成。

## 背景

当前 ClipForge 的初次引导（onboarding）流程存在以下问题：

1. **悬浮面板空间受限**：引导卡片覆盖在列表之上，遮挡了核心功能区域。对于小尺寸面板（如 420x400），引导内容和列表内容相互挤压，用户难以同时看清两者。

2. **引导时机不合理**：用户首次唤起面板时，期望直接使用剪贴板功能，但引导层强制打断了这个流程。

3. **引导内容不完整**：当前引导只覆盖了快捷键操作，缺少权限开启（如辅助功能权限）、采集范围设置、快捷键自定义等关键初始化步骤。

4. **与设置面板职责重叠**：辅助功能权限、快捷键设置等内容既在引导中出现，又在设置面板中存在，导致用户困惑。

5. **借鉴 eco 应用的优秀实践**：eco 等现代 macOS 应用在初始化时，会将用户引导至设置面板完成权限配置和功能介绍，让主界面保持纯净。

## 目标

1. 将初次引导程序从悬浮面板中完全移除，让面板聚焦剪贴板历史、搜索、复制等核心功能。
2. 在设置面板中新增"入门引导"模块，承接权限开启、功能介绍、快捷键设置等初始化流程。
3. 应用启动时，若检测到 `onboardingCompleted` 为 `false`，自动打开设置面板并进入引导流程。
4. 引导流程采用分步卡片形式，清晰展示每个步骤的目标和操作方式。
5. 引导完成后，用户可随时通过设置面板重新查看引导内容。

## 非目标

- 不改变现有的设置面板架构（侧边栏 + 内容区域）。
- 不改变引导文案内容，仅迁移位置和调整展示形式。
- 不改变权限检查和设置保存的后端逻辑。
- 不引入新的引导框架或复杂的状态管理。

## 设计方案

### 整体流程变更

**当前流程：**
```
应用启动 → 用户唤起面板 → 显示引导层 → 用户完成/跳过引导 → 使用核心功能
```

**优化后流程：**
```
应用启动 → 检测 onboardingCompleted → false → 自动打开设置面板并进入引导
                                          → true → 用户正常使用面板
```

### 引导模块结构

设置面板新增"入门引导"分类，包含以下步骤：

```
┌─────────────────────────────────────────┐
│  入门引导                                │
├─────────────────────────────────────────┤
│                                         │
│  [步骤指示器] 1/5                        │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │  欢迎使用 ClipForge               │  │
│  │  简介：快速剪贴板工具             │  │
│  │  核心价值：历史、搜索、快速粘贴   │  │
│  └───────────────────────────────────┘  │
│                                         │
│           [上一步] [跳过] [下一步]       │
│                                         │
└─────────────────────────────────────────┘
```

### 引导步骤设计

#### 步骤 1：欢迎与简介
- **标题**：欢迎使用 ClipForge
- **内容**：介绍 ClipForge 的核心价值（快速剪贴板工具）
- **操作**：了解功能概览

#### 步骤 2：权限开启（核心）
- **标题**：开启辅助功能权限
- **内容**：解释权限用途（读取焦点输入框位置，实现跟随光标定位）
- **操作**：一键跳转到系统设置开启权限，检测权限状态
- **状态**：权限未开启时显示警告，开启后显示成功状态

#### 步骤 3：采集范围设置
- **标题**：配置剪贴板采集
- **内容**：展示可采集的内容类型（文本、HTML、RTF、图片、文件）
- **操作**：开关各类型的采集权限，设置大小上限
- **状态**：显示当前启用的采集类型

#### 步骤 4：快捷键设置
- **标题**：自定义快捷键
- **内容**：展示默认快捷键（Ctrl+V），说明如何修改
- **操作**：录制新的全局快捷键，手动输入快捷键
- **状态**：显示当前配置的快捷键

#### 步骤 5：功能介绍
- **标题**：核心功能速览
- **内容**：以卡片形式展示搜索、收藏、垃圾桶、Agent 等核心功能
- **操作**：完成引导，标记 `onboardingCompleted` 为 `true`

### 设置面板入口

**当前菜单：**
```
ClipForge
---------
🗑 Trash  (T)
⚙ Settings  (,)
? Onboarding (?)
```

**优化后菜单：**
```
ClipForge
---------
🗑 Trash  (T)
⚙ Settings  (,)
```

移除独立的 Onboarding 入口，引导功能整合到设置面板中。

### 视觉设计

1. **引导卡片样式**：
   - 采用玻璃态效果，与设置面板风格一致
   - 卡片宽度自适应内容区域
   - 步骤指示器使用小圆点，当前步骤高亮

2. **权限状态展示**：
   - 未开启：红色警告图标 + 说明文字
   - 已开启：绿色勾选图标 + 成功提示

3. **操作按钮样式**：
   - 主按钮（下一步/完成）：深色背景 + 白色文字
   - 辅助按钮（上一步/跳过）：浅色背景 + 深色文字
   - 权限开启按钮：强调色背景

4. **动画效果**：
   - 步骤切换使用淡入淡出过渡
   - 权限状态变化使用脉冲动画
   - 按钮悬停使用轻微缩放效果

## 交互设计

### 1. 自动触发引导
- 应用启动时，检测 `onboardingCompleted` 状态
- 若为 `false`，自动打开设置面板并切换到"入门引导"分类
- 引导面板自动聚焦到第一个可交互元素

### 2. 步骤导航
- **上一步**：返回上一个步骤（第一步时禁用）
- **跳过**：直接标记引导完成，关闭设置面板
- **下一步**：进入下一个步骤（最后一步时变为"完成"）
- **键盘导航**：支持左右方向键切换步骤，Enter 键确认

### 3. 权限开启流程
- 点击"开启权限"按钮，调用 `request_accessibility_permission`
- 系统设置窗口打开后，设置面板显示"请在系统设置中开启权限"提示
- 用户返回设置面板时，自动检测权限状态并更新 UI

### 4. 设置联动
- 引导过程中修改的设置（采集范围、快捷键等）实时保存
- 引导完成后，用户在设置面板中修改这些设置时，引导不会重新触发

### 5. 重新查看引导
- 用户可在设置面板的"入门引导"分类中随时重新查看引导内容
- 重新查看时，跳过权限检测步骤（若已开启），直接进入功能介绍

## 用户价值

1. **更纯净的面板体验**：悬浮面板不再被引导层遮挡，用户可以直接使用核心功能。

2. **更完整的初始化流程**：引导流程包含权限开启、采集设置、快捷键配置等关键步骤，减少用户后续在设置面板中的摸索。

3. **更合理的空间利用**：设置面板空间充足，适合展示详细的引导内容和操作控件。

4. **更灵活的引导时机**：用户可以随时在设置面板中重新查看引导内容，而不必依赖首次启动的时机。

5. **更符合现代应用习惯**：借鉴 eco 等优秀应用的初始化引导方式，提升产品质感和用户体验。

## 测试要点

1. **首次启动测试**：应用首次启动时，自动打开设置面板并进入引导流程。
2. **引导流程测试**：依次完成五个步骤，每个步骤的内容和操作正常。
3. **权限开启测试**：点击"开启权限"按钮，系统设置窗口打开，返回后权限状态正确更新。
4. **设置联动测试**：引导过程中修改采集范围和快捷键，设置实时保存。
5. **完成后测试**：引导完成后，重新启动应用，不再自动打开设置面板。
6. **重新查看测试**：在设置面板中进入"入门引导"分类，可正常查看引导内容。
7. **面板功能测试**：悬浮面板的历史、搜索、复制等核心功能正常使用。
8. **快捷键测试**：全局快捷键和引导中配置的快捷键正常工作。

## 风险与缓解

1. **用户可能跳过引导**：用户可能直接点击"跳过"按钮，导致权限未开启。缓解：权限开启步骤设置为必须完成（跳过按钮禁用），或在面板使用时提示权限缺失。

2. **设置面板打开时机**：应用启动时自动打开设置面板可能干扰用户。缓解：只在首次启动时触发，且引导完成后不再自动打开。

3. **样式冲突**：新增的引导组件样式可能与现有设置面板样式冲突。缓解：使用语义化 CSS 类名，按域拆分样式。

4. **权限检测延迟**：系统设置返回后，权限检测可能存在延迟。缓解：添加刷新按钮，允许用户手动触发权限检测。

## 与现有提案的关系

1. **与 `settings-interface-redesign` 提案的关系**：本提案依赖设置面板的架构，但不依赖其彻底重构的实现。引导模块可以在现有设置面板架构上实现，后续可随设置面板重构一起迁移。

2. **与 `top-nav-optimization` 提案的关系**：本提案移除菜单中的 Onboarding 入口，与该提案的菜单精简方向一致。

3. **与 `settings-service-unified-protocol` 提案的关系**：本提案使用现有的设置读写逻辑，与该提案的统一服务协议并行推进。
