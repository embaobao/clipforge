# 提案：底部导航迁移至顶部 + 界面精简优化

## 优先级

P1。该优化直接改善用户体验：用户潜意识认为窗口拖拽区域在顶部，且底部导航占用了列表的可视空间。同时将低频使用的垃圾桶收进菜单，让主界面更简洁。

## 背景

当前 ClipForge 浮动面板的布局存在以下问题：

1. **拖拽区域与导航分离**：顶部有一个独立的 `drag-strip` 用于窗口拖拽，但用户潜意识期望在顶部标题栏/工具栏区域就能拖拽窗口。底部 `BottomDock` 使用 `data-tauri-drag-region` 支持拖拽，但位置不符合用户直觉。

2. **底部导航浪费空间**：`BottomDock` 固定占用 48px 高度（`--cf-dock-height`），对于一个悬浮面板来说，底部空间应该优先留给列表内容，而不是导航。

3. **垃圾桶入口过于突出**：回收站（trash）作为低频操作，与历史记录、收藏夹并列在底部导航，视觉权重过高，干扰主工作流。

4. **顶部空间闲置**：搜索栏上方有空白区域，未充分利用。

5. **Dock 折叠体验不佳**：当前支持滚动时隐藏底部 Dock，但恢复时的动画不够流畅，且折叠状态下用户无法快速切换视图。

## 目标

1. 将底部导航（视图切换、Agent、设置菜单）整体迁移到顶部，与搜索栏合并为统一的顶部工具栏。
2. 将垃圾桶（trash）入口移至设置菜单中，作为二级选项，减少主界面视觉干扰。
3. 顶部工具栏支持窗口拖拽，符合用户直觉。
4. 移除底部 `BottomDock`，增加列表可视区域高度。
5. 优化工具栏交互细节，保持功能不变的前提下提升体验。

## 非目标

- 不改变视图切换的功能逻辑（history/favorites/trash 的数据模型不变）。
- 不改变快捷键映射（历史记录、收藏、垃圾桶的快捷键保持不变）。
- 不改变多选择模式的交互流程。
- 不引入新的视图或页面。

## 设计方案

### 布局结构变更

**当前布局：**
```
┌─────────────────────────────────┐
│  drag-strip (仅拖拽)            │
├─────────────────────────────────┤
│  [搜索栏]                        │
├─────────────────────────────────┤
│                                 │
│         列表区域                 │
│                                 │
├─────────────────────────────────┤
│  [History] [Favorites] [Trash]  │  ← BottomDock
│  [Agent] [Status] [Settings]    │
└─────────────────────────────────┘
```

**优化后布局：**
```
┌─────────────────────────────────────────────┐
│  [History] [Favorites]    [搜索]    [●●●]   │  ← TopToolbar (支持拖拽)
├─────────────────────────────────────────────┤
│                                             │
│           列表区域 (增加 48px)               │
│                                             │
└─────────────────────────────────────────────┘
```

### 顶部工具栏结构

```
┌─────────────────────────────────────────────────────────────┐
│  [◀ History]  [★ Favorites]    [搜索框]    [⚡ Agent]  [●●●]  │
│   ↑ 视图切换                    ↑ 搜索       ↑ Agent入口 ↑ 菜单  │
└─────────────────────────────────────────────────────────────┘
```

#### 左侧：视图切换
- **History（历史）**：当前激活视图时高亮，图标 + 文字
- **Favorites（收藏）**：图标 + 文字

#### 中间：搜索栏
- 保持现有搜索框样式和功能
- 搜索框宽度自适应，占据中间大部分空间

#### 右侧：操作按钮
- **Agent 入口**：带上下文计数徽章
- **菜单按钮（●●●）**：点击展开下拉菜单

### 菜单内容调整

**当前菜单：**
```
ClipForge
---------
⚙ Settings  (,)
? Onboarding (?)
```

**优化后菜单：**
```
ClipForge
---------
🗑 Trash  (T)
⚙ Settings  (,)
? Onboarding (?)
```

新增 `Trash` 选项，快捷键保持 `T`（与当前一致）。

### 拖拽区域调整

- 整个顶部工具栏（`.top-toolbar`）设置为 `data-tauri-drag-region`
- 工具栏内的按钮区域需要阻止事件冒泡（`stopPropagation`），避免点击按钮时触发拖拽
- 搜索框区域也需要阻止事件冒泡

## 交互优化点

### 1. 视图切换动画
- 保持现有 tab 切换动画（highlight 滑动效果）
- 切换时列表内容带淡入淡出过渡

### 2. 菜单展开方向
- 当前菜单从底部向上展开（`side="top"`）
- 优化后从顶部工具栏向下展开（`side="bottom"`），符合直觉

### 3. 垃圾桶入口
- 从底部导航移除，移入菜单
- 菜单中显示垃圾桶图标和文字，保持快捷键提示 `(T)`
- 在 trash 视图时，菜单中的 trash 项显示勾选状态

### 4. 搜索栏行为
- 保持现有搜索激活/收起逻辑
- 搜索激活时，视图切换按钮缩小为仅图标模式，节省空间

### 5. 滚动行为
- 移除底部 Dock 的折叠逻辑（`isFooterHidden`）
- 列表可以滚动到面板底部，不再被 Dock 遮挡

### 6. 状态显示
- 当前底部状态栏（`StatusLine`）移至顶部工具栏右侧，Agent 按钮旁边
- 状态文字精简为图标 + 简短文字

### 7. 快捷键反馈
- 按 `T` 键直接切换到 trash 视图（保持现有行为）
- 按 `Cmd/Ctrl+,` 打开设置（保持现有行为）

## 视觉细节优化

### 1. 工具栏样式
- 使用玻璃态效果（glass morphism），与搜索栏风格一致
- 工具栏高度：40px（比原 Dock 少 8px，节省空间）
- 顶部圆角：与面板圆角一致

### 2. 视图按钮样式
- 非激活状态：灰色图标 + 浅色文字
- 激活状态：深色背景 + 白色图标 + 文字
- 悬停效果：轻微缩放 + 背景高亮

### 3. 菜单按钮样式
- 极简设计：三个小圆点（●●●）
- 悬停时展开为背景高亮
- 菜单内容使用简洁列表样式

### 4. 搜索栏整合
- 搜索栏与工具栏无缝衔接
- 搜索激活时，工具栏高度自适应

### 5. 列表区域
- 移除底部内边距（原 Dock 占用区域）
- 列表项可以延伸到面板底部边缘
- 增加列表可视区域约 48px

## 技术实现要点

### 组件变更

1. **新建 `TopToolbar` 组件**：
   - 整合视图切换（History/Favorites）
   - 嵌入搜索栏（`GlassSearchBar`）
   - Agent 入口按钮
   - 菜单按钮（含 Trash/Settings/Onboarding）
   - 状态栏

2. **移除 `BottomDock` 组件**：
   - 删除底部导航相关 JSX
   - 删除底部拖拽区域

3. **修改 `App.tsx` 主布局**：
   - 将 `TopToolbar` 放在搜索栏位置或上方
   - 移除 `BottomDock` 引用
   - 调整滚动和布局逻辑

### CSS 变更

1. **新增 `.top-toolbar` 样式**：
   - 玻璃态背景
   - 拖拽区域样式
   - 按钮布局

2. **修改 `.content-column` 布局**：
   - 移除底部 Dock 相关间距
   - 调整列表滚动区域

3. **删除 `.list-footer` 相关样式**：
   - 移除底部导航所有样式定义

4. **调整 `.toolbar` 样式**：
   - 搜索栏整合到顶部工具栏

### 状态管理

1. **`isFooterHidden`**：移除，不再需要
2. **`isSearchCompact`**：保留，用于搜索激活时的工具栏收缩
3. **视图切换状态**：保持现有 `activeView` 逻辑

## 用户价值

1. **更符合直觉的拖拽体验**：用户可以在顶部工具栏区域拖拽窗口，无需寻找单独的拖拽条。

2. **更多可视内容**：移除底部 Dock 后，列表区域增加约 48px，在小面板中尤为明显。

3. **更简洁的界面**：垃圾桶作为低频操作收进菜单，减少主界面视觉干扰。

4. **更清晰的信息层级**：视图切换、搜索、操作按钮在顶部形成统一的功能带，结构更清晰。

5. **保持功能完整**：所有现有功能和快捷键保持不变，用户无需重新学习。

## 风险与缓解

1. **用户习惯改变**：用户可能需要适应新的导航位置。缓解：保持视图切换按钮的视觉风格和位置逻辑（左侧视图切换，右侧操作）。

2. **拖拽冲突**：按钮点击可能误触发拖拽。缓解：所有可交互元素使用 `stopPropagation` 阻止事件冒泡。

3. **响应式问题**：窄面板下工具栏元素可能拥挤。缓解：搜索激活时视图按钮缩小为图标模式，菜单按钮始终显示。

4. **样式冲突**：现有 `.toolbar` 和 `.list-footer` 样式可能与新布局冲突。缓解：按域拆分样式，使用 P-FINAL 覆盖确保样式优先级。
