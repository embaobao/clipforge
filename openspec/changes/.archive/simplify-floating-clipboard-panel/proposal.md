# 提案：简化高密度悬浮剪贴板面板

## 背景

现有提案已经覆盖了 ClipForge 的基础剪贴板能力、NSPanel 悬浮窗口、数字快捷键、监听升级和 UI 风格升级，但这些提案之间存在体验重心不一致：

- `design-style-upgrade` 偏向完整管理窗口和三栏信息架构，容易把快捷面板做成管理后台。
- `nspanel-floating-panel` 强调悬浮、轻量、快速唤起，更符合 ClipForge 当前第一目标。
- `numeric-shortcut-paste` 要求键盘和数字快捷复制优先，但当前列表行信息密度不足。
- `clipboard-listener-upgrade` 要求状态反馈和写回防护，状态应服务于复制流，不应抢占顶部空间。

本变更将这些提案统一为一个当前阶段的产品方向：**ClipForge 主面板首先是高密度、低干扰、可键鼠快速复制、可扩展到 Agent/AI 的悬浮剪贴板面板**。

这不是一次单纯视觉改版。核心价值来自四条能力主线：

1. 悬浮窗快捷性：窗口唤起、定位、激活、焦点和输入区域必须可检测、可恢复、可验证。
2. 键盘极速路径：搜索输入、方向选择、数字快捷、复制回写和多选聚合必须低延迟、不中断。
3. 内容类型扩展：文本、链接、图片、文件、表格/图表等内容要有统一展示、预览、打开和复制抽象。
4. Agent/AI 入口预留：当前仍是剪贴板工具，但状态、内容模型、复制动作和插件接口要为后续 Agent 稳定对接保留边界。

## 目标

- 移除紧凑面板右侧栏，把历史、收藏、垃圾箱、设置收到底部操作栏。
- 搜索区采用半透明液态玻璃效果，覆盖感弱，不阻断列表浏览。
- 列表主信息改为一行原始剪贴板内容，中间省略，提升内容感知和密度。
- 数字块作为快捷视觉锚点和多选入口：行主体点击默认复制，数字块点击进入/切换多选；键盘数字输入默认服务搜索，`Cmd+1` 到 `Cmd+9` 才作用于对应可见项。
- 单条预览改为顶部固定区域，开启时固定高度，不因内容变化抖动。
- 多选改为底部动态悬浮操作栏，展示聚合复制结果预览，而不是选中卡片列表。
- 状态信息收到底部，刷新按钮取消，采集刷新归并到状态和自动监听。
- 保留收藏、删除、垃圾箱恢复链路，但减少行内操作露出。
- 增加快捷聚焦动效：鼠标 hover、键盘选中和即将复制项共享一个目标光标/聚焦框。
- 建立 shadcn 语义 token + ClipForge 产品 token + 液态玻璃/像素动效 token 的完整风格系统。
- 明确悬浮窗激活检测、输入焦点、键盘选择性能、内容类型适配和 Agent/插件扩展的框架约束。

## 非目标

- 本阶段不引入 TanStack Router。历史、收藏、垃圾箱仍属于同一悬浮面板状态，不需要 URL 路由和 routeTree。
- 不新增重型 UI 库。
- 不把 AI/MCP 配置暴露到当前主面板。
- 不实现复杂多页面管理后台。
- 不把 Agent/AI 能力前置为主体验；只规划稳定接入边界和未来扩展入口。

## 用户价值

- 一屏显示更多剪贴板内容，减少滚动。
- 默认点击即可复制，数字快捷和键盘导航更清晰。
- 多选时能直接看到最终聚合复制文本，降低误复制风险。
- 垃圾箱保留恢复能力，但不干扰日常复制。
- 悬浮面板更像快捷工具，而不是数据管理控制台。
- 窗口唤起后输入焦点稳定，用户可以立即搜索、选择、复制。
- 后续支持图片、文件、表格/图表、插件动作和 Agent 工具调用时，不需要推翻主面板结构。

## 旧提案整合结论

- 延续 `nspanel-floating-panel` 的悬浮轻量原则。
- 延续 `numeric-shortcut-paste` 的数字快捷和键盘优先原则，但视觉上把数字块变为主交互锚点。
- 延续 `design-style-upgrade` 的高密度、毛玻璃和语义 token 方向，暂不采用三栏管理窗口作为主体验。
- 延续 `clipboard-listener-upgrade` 的状态反馈要求，但状态应下沉到底部，不占顶部搜索/列表空间。
