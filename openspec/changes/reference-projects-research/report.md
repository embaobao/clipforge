## 二、各项目核心特征与借鉴价值

### 2.1 Maccy — macOS 原生剪贴板标杆

**定位**：极致轻量的纯 macOS 原生剪贴板管理器，MIT 许可证，~3MB 安装包。

**技术亮点**：

1. **剪贴板轮询策略**：Timer 轮询 `NSPasteboard.general.changeCount`，默认 500ms（可调至 100ms），比通知机制更可靠
2. **5 种面板定位策略**：cursor / statusItem / window / center / lastPosition，其中 lastPosition 用归一化坐标（0~1 比例）存储，跨分辨率适配
3. **NSPanel 浮动窗口**：`.statusBar` level + `.nonactivatingPanel` styleMask，不抢焦点、浮动在所有窗口上方
4. **粘贴模拟**：通过 CGEvent 模拟 ⌘V，支持 Dvorak-QWERTY 键盘布局
5. **安全过滤体系**：自动忽略 `ConcealedType`（密码管理器）、`TransientType`（临时数据）、`AutoGeneratedType`；支持正则过滤和按应用忽略
6. **固定条目**：⌥+P 固定后分配永久数字快捷键，清除历史不影响固定项

**对 ClipForge 的借鉴**：

| 借鉴点 | 说明 | 优先级 |
|--------|------|--------|
| 归一化坐标存储窗口位置 | 跨分辨率/多显示器通用方案，解决当前 AppleScript 异步定位的脆弱性 | 高 |
| NSPanel 浮动面板属性配置 | `.nonactivatingPanel` + `.statusBar` level，让面板不抢焦点 | 高 |
| 安全过滤类型体系 | 识别 ConcealedType/TransientType，不采集密码管理器内容 | 中 |
| 固定条目 + 数字快捷键 | 固定后自动分配数字键，⌘+数字直达粘贴 | 中 |
| vim 风格快捷键 | ^J/^K/^P/^N 导航，服务键盘用户 | 低 |

### 2.2 EcoPaste — 最接近 ClipForge 架构的 Tauri v2 项目

**定位**：Tauri v2 + Rust 后端主导的剪贴板管理器，macOS + Windows 双平台。

**技术亮点**：

1. **clipboard-rs fork 方案**：fork 了 clipboard-rs 加入 `new_with_interval` 接口，macOS 轮询间隔从默认 500ms 调低至 120ms，提升跟手性
2. **WritebackGuard 写回防护**：标记自身粘贴操作为 `suppress`，检测 `should_skip` 防止回环入库，比简单布尔标记更可靠
3. **blake3 哈希去重**：`content_hash = blake3(kind:content)`，重复内容只累加 `use_count` 不新增行
4. **SQLite WAL + FTS5**：WAL 日志模式 + Normal 同步 + FTS5 全文搜索，不是内存 filter 或 SQL LIKE
5. **窗口定位三策略**：FollowCursor（对齐光标） / Center（居中当前显示器） / Remember（恢复上次位置，含越界 fallback）
6. **NSPanel (tauri-nspanel)**：macOS 上使用 NSPanel 实现真正悬浮面板，异步显示
7. **DestroyWhenIdle 生命周期**：次级窗口空闲超时后自动销毁，按需重建，节省内存
8. **敏感内容正则检测**：内置 PEM 私钥、GitHub/OpenAI/Slack token、AWS Access Key、JWT 检测

**对 ClipForge 的借鉴**：

| 借鉴点 | 说明 | 优先级 |
|--------|------|--------|
| WritebackGuard 写回防护 | 当前 ClipForge 缺少回环防护，自身粘贴操作会重复入库 | 高 |
| blake3 哈希去重 | 替代当前 localStorage 的简单去重逻辑 | 高 |
| SQLite WAL + FTS5 搜索 | 从 localStorage 迁移到 SQLite 的完整参考方案 | 高 |
| tauri-nspanel 悬浮面板 | 替代当前 AppleScript + WebviewWindow 的定位方案 | 高 |
| 窗口定位三策略 | FollowCursor/Center/Remember 覆盖主流场景 | 高 |
| DestroyWhenIdle 窗口生命周期 | 设置窗口等次级窗口空闲销毁 | 中 |
| 敏感内容正则检测 | 安全层能力 | 中 |
| Valtio settings mirror 前后端状态同步 | 设置读写模式参考 | 低 |

### 2.3 TieZ Clipboard — 功能最丰富的 Tauri v2 剪贴板工具

**定位**：功能大而全的剪贴板工具，包含云同步、局域网传输、隐私脱敏等。

**技术亮点**：

1. **贴边收纳（Edge Docking）**：窗口自动吸附屏幕边缘隐藏，鼠标移入呼出——独特的交互模式
2. **多端实时同步**：同时支持 WebDAV 和 MQTT 协议，验证码秒传是独创场景
3. **局域网文件传输**：内嵌 axum HTTP 服务器，无需外网
4. **隐私脱敏**：自动识别身份证号、手机号、邮箱，预览打码 + 加密存储
5. **顺序粘贴（Sequential Paste）**：PasteQueue 状态管理，逐步粘贴模式
6. **Win+数字快速粘贴**：Windows 端底层钩子实现系统级快捷粘贴
7. **外部编辑器协作**：一键用外部编辑器修改内容，存盘后自动写回
8. **5 款主题风格**：不仅是颜色切换，而是完整视觉风格

**对 ClipForge 的借鉴**：

| 借鉴点 | 说明 | 优先级 |
|--------|------|--------|
| 贴边收纳交互 | 独特的窗口交互模式，后续可考虑 | 低（非第一阶段） |
| 隐私脱敏 | 预览打码 + 加密存储 | 低（非第一阶段） |
| 顺序粘贴 | 高频办公场景 | 低（非第一阶段） |
| 分层架构 (domain/services/infrastructure) | Rust 端代码组织参考 | 中 |
| 多显示器位置映射 | `remap_fixed_window_position` 在多显示器间按比例映射 | 中 |

> 注：TieZ 功能虽多但部分实现质量一般，且 AGPLv3 许可证有传染性，**不宜直接引用代码**，仅做设计参考。

### 2.4 Power Paste — 交互创新的 Tauri v2 剪贴板工具

**定位**：Vue 3 + Tauri v2 剪贴板工具，核心差异化在"快速粘贴"交互。

**技术亮点**：

1. **按住快捷键循环粘贴**：按住快捷键 → 面板弹出 → 继续按键循环切换候选项 → 松手即粘贴——极高效的交互范式
2. **双层剪贴板后端**：Windows 用 Win32 原生 API（CF_UNICODETEXT/HTML Format/PNG），其他平台 fallback 到 Tauri 插件
3. **目标感知粘贴（target-aware paste）**：记录上次焦点窗口，直接粘贴回目标应用，比"仅回写剪贴板"少一步操作
4. **Finder 式颜色标签**：7 色 × 3 标签/条目 + 自定义标签名，比简单收藏更有组织力
5. **Payload 降级链**：Mixed → RichText → Html → Text → Empty，逐级降级保证可用性
6. **手机互传**：二维码 + 浏览器，零安装成本
7. **应用忽略列表**：多维度匹配（路径/BundleID/进程名/显示名）

**对 ClipForge 的借鉴**：

| 借鉴点 | 说明 | 优先级 |
|--------|------|--------|
| 按住快捷键快速粘贴 | 创新交互范式，极大提升高频粘贴效率 | 高（核心体验） |
| Payload 降级链 | 剪贴板内容解析的健壮性策略 | 高 |
| 目标感知粘贴 | 从"复制回剪贴板"升级到"直接粘贴到目标窗口" | 高（第二阶段） |
| Finder 式颜色标签 | 比当前简单收藏更有组织力 | 中 |
| 应用忽略列表 | 解决密码管理器等敏感场景 | 中 |
| SQLite 单一数据源 + 前端事件驱动 | 前端不替代 SQLite 的权威地位 | 高 |

### 2.5 tauri-plugin-positioner — Tauri 官方窗口定位插件

**定位**：Tauri 官方维护的轻量级窗口定位插件。

**能力**：
- 15 种预定义位置：9 种屏幕相对 + 6 种托盘相对
- 托盘定位：`TrayLeft/TrayRight/TrayCenter/TrayBottomLeft/TrayBottomRight/TrayBottomCenter`
- v2 新增 `move_window_constrained`：对 Tray 类位置做屏幕边界约束

**局限性**：
- **不支持鼠标光标定位**：只有屏幕和托盘两类锚点
- **只支持预定义位置**：不支持自定义偏移量
- **托盘位置依赖事件捕获**：必须先有 TrayIconEvent 才能定位，否则返回错误
- **多托盘场景**：只保存最近一次事件的位置
- **高 DPI 处理有限**：scale factor = 1.0 硬编码
- **无动画过渡**

**对 ClipForge 的借鉴**：

| 借鉴点 | 说明 | 优先级 |
|--------|------|--------|
| 托盘图标旁弹出窗口 | 替代当前 AppleScript 定位方案的基础能力 | 高 |
| Tray* 位置枚举 | 托盘点击唤起面板的标准做法 | 高 |
| move_window_constrained | 防止窗口超出屏幕 | 中 |

> 注：该插件**只能解决托盘点击弹出场景**，不能解决"跟随输入框光标弹出"的需求。光标跟随需要 EcoPaste 的 FollowCursor 方案或 Maccy 的 cursor 定位方案。

## 三、关键技术决策对比

### 3.1 剪贴板监听方案

| 项目 | 方案 | 轮询间隔 | 跨平台 |
|------|------|----------|--------|
| Maccy | Timer 轮询 changeCount | 500ms（可调 100ms） | macOS only |
| EcoPaste | clipboard-rs fork | 120ms | macOS + Windows |
| TieZ | objc2-app-kit / clipboard-rs / arboard | 未明确 | macOS + Windows |
| Power Paste | tauri-plugin-clipboard-next + Win32 原生 | 未明确 | macOS + Windows + Linux |

**建议**：ClipForge 当前使用 Tauri 插件方案，后续迁移到 EcoPaste 的 clipboard-rs fork 方案（120ms 间隔），或参考 Power Paste 的双层后端策略（macOS 用原生 API，其他平台 fallback 插件）。

### 3.2 窗口定位方案

| 项目 | 方案 | 支持光标跟随 | 支持托盘定位 | 支持多显示器 |
|------|------|-------------|-------------|-------------|
| Maccy | 5 种策略 (cursor/statusItem/window/center/lastPosition) | ✅ | ✅ (NSStatusItem) | ✅ |
| EcoPaste | 3 种策略 (FollowCursor/Center/Remember) | ✅ | ❌ | ✅ |
| TieZ | 贴边收纳 + 跟随鼠标 | ✅ | ❌ | ✅ (比例映射) |
| Power Paste | 未明确 | ❌ | ❌ | 未明确 |
| tauri-plugin-positioner | 15 种预定义位置 | ❌ | ✅ | 部分 |

**建议**：ClipForge 应组合使用：
1. **tauri-plugin-positioner** 解决托盘点击弹出（TrayCenter）
2. **自研 FollowCursor** 解决跟随输入框光标弹出（参考 EcoPaste 的 position.rs）
3. **归一化坐标 Remember** 解决记住上次位置（参考 Maccy 的 PopupPosition.swift）

### 3.3 数据持久化方案

| 项目 | 方案 | 搜索能力 | 去重机制 |
|------|------|----------|----------|
| Maccy | SwiftData (SQLite) | 模糊搜索 | changeCount 检测 |
| EcoPaste | SQLite (sqlx) + WAL + FTS5 | 全文搜索 | blake3 哈希 |
| TieZ | SQLite (rusqlite) | 全文搜索 + tag:语法 | DefaultHasher + 像素哈希 |
| Power Paste | SQLite (rusqlite) | 全文搜索 | SHA256 |

**建议**：ClipForge 从 localStorage 迁移到 SQLite 时：
- 使用 **sqlx** 驱动（EcoPaste 方案，编译期 SQL 检查）
- 启用 **WAL 模式 + FTS5 全文搜索**
- 去重使用 **blake3 哈希**（性能优于 SHA256）

### 3.4 粘贴方式

| 项目 | 方案 | 用户体验 |
|------|------|----------|
| Maccy | CGEvent 模拟 ⌘V | 最直接，但需 Accessibility 权限 |
| EcoPaste | 写回剪贴板 + 可选模拟粘贴 | 安全但有额外步骤 |
| TieZ | 写回剪贴板 + 模拟粘贴 | 同上 |
| Power Paste | 目标感知粘贴 | 记住焦点窗口，直接粘贴回去 |

**建议**：ClipForge 第一阶段先"写回剪贴板"，第二阶段实现"目标感知粘贴"（Power Paste 方案），第三阶段可选 CGEvent 级别的模拟粘贴。

## 五、针对 ClipForge 的提案

### 提案 1：剪贴板监听与采集升级 (P1)

**问题**：当前剪贴板监听基于 Tauri 插件，无写回防护、无去重哈希、不支持图片和文件类型。

**方案**：
1. 引入 **WritebackGuard**（参考 EcoPaste）：自身粘贴操作标记 `suppress`，采集时检测 `should_skip` 防止回环
2. 引入 **blake3 内容哈希去重**（参考 EcoPaste）：`content_hash = blake3(kind:content)`，重复内容只累加计数
3. 扩展采集类型：图片（PNG/JPEG）、文件路径、HTML 富文本
4. 参考 Power Paste 的 **Payload 降级链**：Mixed → RichText → Html → Text → Empty

**预期效果**：消除了回环重复问题，支持更多内容类型，去重准确性从"字符串比较"升级到"哈希指纹"。

---

### 提案 2：从 localStorage 迁移到 SQLite (P1)

**问题**：当前使用 localStorage 持久化，无法支持全文搜索、大量历史、事务一致性。

**方案**（参考 EcoPaste 的 SQLite 方案）：
1. 使用 **sqlx** 驱动（编译期 SQL 检查、async）
2. 启用 **WAL 日志模式** + Normal 同步 + 外键约束
3. 启用 **FTS5 全文搜索**：中文/英文混合搜索
4. 图片存磁盘文件系统（`resources/clipboard-images/`），数据库存路径引用
5. 分页查询（默认 20 条/页），返回 total + has_more

**数据模型参考**：
```sql
-- 核心表
CREATE TABLE clipboard_items (
  id TEXT PRIMARY KEY,
  kind TEXT NOT NULL,          -- 'text' | 'image' | 'files' | 'html'
  content_hash TEXT NOT NULL,  -- blake3 哈希
  content TEXT,                -- 文本内容或图片路径
  search_text TEXT,            -- FTS5 搜索专用
  summary TEXT,                -- 摘要（≤512字符），列表渲染用
  source_app TEXT,             -- 来源应用 bundle ID
  is_pinned INTEGER DEFAULT 0,
  is_favorite INTEGER DEFAULT 0,
  use_count INTEGER DEFAULT 1,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

-- FTS5 虚拟表
CREATE VIRTUAL TABLE clipboard_items_fts USING fts5(search_text, content=clipboard_items, content_rowid=rowid);
```

### 提案 3：窗口定位方案升级 (P1)

**问题**：当前使用 AppleScript 异步获取输入框位置，存在卡顿和脆弱性问题。

**方案**（组合方案）：
1. **引入 tauri-plugin-positioner**：解决托盘点击弹出场景（`Position.TrayCenter`）
2. **自研 FollowCursor 定位**（参考 EcoPaste 的 position.rs）：
   - 获取光标所在显示器
   - 窗口左上角对齐光标位置
   - 不超出显示器边界
3. **归一化坐标 Remember**（参考 Maccy）：
   - 存储为相对于可见区域的 [0,1] 比例坐标
   - 跨分辨率/多显示器适配
4. **多显示器适配**（参考 TieZ 的 `remap_fixed_window_position`）：
   - 窗口位置在显示器切换时自动重映射

**定位策略枚举**：
```rust
enum PanelPosition {
  TrayCenter,       // 托盘图标下方居中
  FollowCursor,     // 跟随鼠标光标
  Center,           // 居中当前显示器
  LastPosition,     // 恢复上次位置（归一化坐标）
}
```

### 提案 4：快速粘贴交互 (P1/P2)

**问题**：当前只能"复制回剪贴板"，用户需手动切换窗口粘贴。

**方案**：
1. P1 — 数字快捷键：面板内 ⌘+1~0 直接粘贴前 10 条可见条目（参考 Maccy/Power Paste）
2. P2 — 按住快捷键循环粘贴（参考 Power Paste）：按住快捷键 → 面板弹出 → 继续按键循环切换 → 松手即粘贴
3. P2 — 目标感知粘贴（参考 Power Paste）：记录上次焦点窗口，直接粘贴回目标应用

### 提案 5：安全过滤体系 (P2)

**问题**：当前无敏感内容过滤，密码管理器复制的内容会被采集。

**方案**（参考 Maccy + EcoPaste）：
1. macOS 端检查 `ConcealedType` / `TransientType` / `AutoGeneratedType`（Maccy 方案）
2. 内置正则检测 PEM 私钥、Token、AWS Key 等（EcoPaste 方案）
3. 可配置的忽略应用列表（Power Paste 方案）
4. 可配置的正则过滤规则（Maccy 方案）

### 提案 6：NSPanel 悬浮面板 (P1)

**问题**：当前 WebviewWindow 作为面板不够原生，焦点管理不稳定。

**方案**（参考 EcoPaste 的 tauri-nspanel）：
1. 引入 `tauri-nspanel` crate，macOS 上使用 NSPanel 实现
2. 配置 NSPanel 属性：
   - `.nonactivatingPanel`：不抢焦点
   - `.statusBar` level：浮动在普通窗口上方
   - `hidesOnDeactivate = false`：失焦不自动隐藏（手动控制）
3. Windows 端继续使用 WebviewWindow + `always_on_top`

## 七、风险提示

1. **许可证兼容性**：EcoPaste 和 Power Paste 均为 AGPLv3，**不可直接复制代码**，只能参考设计思路和独立实现。Maccy (MIT) 和 tauri-plugin-positioner (Apache-2.0/MIT) 可安全引用。
2. **tauri-nspanel 维护性**：该 crate 是社区维护，需评估其与 Tauri v2 最新版本的兼容性。
3. **FTS5 中文分词**：SQLite FTS5 默认 tokenizer 对中文支持有限（按字符分词），若需更精确的中文分词，需引入 jieba 等分词器或使用 simple tokenizer + ngram。
4. **Accessibility 权限**：模拟粘贴（CGEvent）在 macOS 上需要辅助功能权限，需在 UI 中引导用户授权。
