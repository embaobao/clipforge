# 提案：稳定黑白高密度快捷面板

## 背景

当前快捷面板已经做过多轮布局调整，但仍存在三个直接影响验证的问题：

- 启动验证容易命中旧的生产 bundle，导致看到 `主导航`、常驻筛选 chip、右侧 rail 等旧布局。
- 样式覆盖层过多，黑白主题、虚线选中、行级密度和底部操作区没有形成唯一视觉规则。
- 多选模式缺少清晰的顶部导航/操作台，用户无法从界面结构上判断当前处于多选、聚合或详情下钻。

本提案只处理主快捷面板的视觉与交互收敛，不扩展 AI、插件、设置页或数据层能力。

## 目标

1. 主面板采用黑白灰主题，禁止默认蓝色、青色、橙色等非语义强调色进入选中态。
2. 选中态使用黑色直角虚线边框，不再使用歪斜移动目标框、彩色角标或动画 pulse。
3. 多选模式必须在顶部显示导航/操作台：返回、当前模式、选中数量、聚合详情入口。
4. 列表行保持高密度：一行内容、8px 内边距、行内默认不暴露删除，收藏状态可见但不抢内容空间。
5. 启动验证必须只保留 dev 应用进程，不能让 `target/debug/bundle/macos/ClipForge.app` 旧包干扰判断。

## 非目标

- 不在本提案中重做设置页。
- 不在本提案中新增插件系统实现。
- 不在本提案中重新设计数据库、剪贴板采集或原生定位算法。
- 不再引入复杂选中动画；如果 CSS 无法保持稳定和对齐，选中动效直接删除。

## 验收标准

- 运行应用后，主面板不出现 `主导航`、`快速筛选`、`保存为筛选规则`、右侧 rail。
- 选中行只显示黑色直角虚线边框；无青色、蓝色或歪斜角标。
- 多选模式顶部出现黑白玻璃操作台，包含返回、多选路径、选中数量、聚合入口。
- 底部 dock 只保留状态、历史、收藏、垃圾箱、设置，不直接暴露删除。
- 多选删除只在多选上下文操作区出现。
- 验证前进程列表中不得存在 `src-tauri/target/debug/bundle/macos/ClipForge.app/Contents/MacOS/clipforge`。
