# 提案：构建跨平台剪贴板工具

## 背景

Clipy 的菜单式交互适合轻量选择。ClipForge 第一阶段先保持这类快速体验，补齐跨平台剪贴板历史、片段、文件夹、快捷唤起和复制回写；搜索、归档和批量操作作为工具增强，不提前扩成平台叙事。

## 目标

- 新建跨平台项目，保持 Clipy 核心功能一致。
- 使用真实窗口承载搜索、列表、详情和批量操作，同时保留快速菜单优先。
- 提供更清晰的本地架构，为归档、搜索增强和 MCP 标准工具接口预留扩展点。
- 优先性能和输入稳定性，避免菜单重建导致焦点跳动。

## 用户价值

- 搜索框常驻，输入后直接过滤并展示结果。
- 历史、片段和文件夹视图一致，优先服务快速粘贴。
- 可以复制、收藏、归档、删除和全选当前结果删除。
- 后续可把剪贴板作为 MCP 工具可调用的本地数据源。

## 范围

- Tauri v2 桌面应用
- React 剪贴板工具 UI
- Rust 原生剪贴板读写命令
- localStorage 持久化 MVP
- OpenSpec 与项目规则初始化

## 后续增量

- SQLite 持久化
- 后续本地语义索引
- 全局快捷键与系统托盘
- MCP server：search/copy/archive/delete
- 跨设备同步和隐私规则
